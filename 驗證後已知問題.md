### 1. 側欄縮窄

- **現況**：[sidebar.tsx](web/src/components/sidebar.tsx) 寬度 `w-60` (240px)，文字「個股分析」只佔約一半，右側大片空白
- **修改**：縮到約 `w-36` ~ `w-40` (144~160px)，剛好容納圖示 + 4 字中文 + padding
- **使用者確認**：**保留圖示**（icon 左、文字右的既有配置）
- **連動**：主內容 (`<main>`) 自動填補釋出的空間，整體右移

### 2. OHLC 欄位中文化 + 右欄上移

- **現況**：HeaderRow 顯示 7 欄 `O / H / L / PC / V / 買量 / 賣量`，前 5 個是英文縮寫；右欄（技術分析總覽等）起點偏低
- **修改**：
  - 標籤翻譯（**使用者確認**）：
    - `O → 開盤` ／ `H → 最高` ／ `L → 最低` ／ `PC → 前收` ／ `V → 成交量`
  - 「買量 / 賣量」兩欄 **盤後隱藏、盤中保留**（用 [RealtimeQuote.is_market_open](web/src/types/analysis.ts) 判斷；盤中行為等使用者實際盤中見到後再評估，**列入後續討論**）
  - 整列左移、右側內縮（目前太右）
- **右欄上移**：技術分析總覽 / 關鍵價位 / 籌碼分析三個 panel 的頂部與 **QuantTrader logo / 個股分析 標題** 對齊（不是與 OHLC 列對齊），右欄盡量緊湊
- **影響**：[dashboard-page-client.tsx](web/src/components/dashboard/dashboard-page-client.tsx) 的 `HeaderRow` + 主版 grid 結構

### 3. MA 線細化 + 標籤上色

- **現況**：[candlestick-chart.tsx:178, 184, 190](web/src/components/dashboard/candlestick-chart.tsx) MA5/MA20/MA60 用 `lineWidth: 2`，K 棒容易被遮擋；右上角文字「MA5 / MA20 / MA60」是純白色
- **修改**：
  - 三條 MA 線 `lineWidth: 2 → 1`
  - 文字顏色對應線色：**MA5 = 橙 `#f59e0b`**，**MA20 = 藍 `#3b82f6`**，**MA60 = 紫 `#c084fc`**
- **影響**：[candlestick-chart.tsx](web/src/components/dashboard/candlestick-chart.tsx)、[dashboard-page-client.tsx](web/src/components/dashboard/dashboard-page-client.tsx)（標籤位置）

### 4. 市場切換 active state

- **現況**：[market-switcher.tsx](web/src/components/market-switcher.tsx) 兩顆 button 視覺上看不出哪個被選中
- **修改**：選中狀態加背景色 + 對比文字色，例如 `bg-slate-100 text-slate-900` vs `text-slate-400`
- **影響**：[market-switcher.tsx](web/src/components/market-switcher.tsx)

### 5. K 線 hover crosshair tooltip

- **現況**：lightweight-charts 預設只在 price scale 顯示游標位置的價格，不顯示完整 OHLC + MA
- **修改**：使用 `chart.subscribeCrosshairMove()` API 自訂浮動 tooltip
  - 顯示欄位：**日期 / 開盤 / 最高 / 最低 / 收盤 / 成交量 / MA5 / MA20 / MA60**
  - 行為（**使用者確認**）：**跟著游標移動**，避免視線跨越畫面
  - 樣式：半透明深色背景、圓角、邊框、monospace 字
  - hover 離開時自動隱藏
- **影響**：[candlestick-chart.tsx](web/src/components/dashboard/candlestick-chart.tsx)（新增 tooltip DOM + crosshair listener）

### 6. 成交量 hover 顯示

- **修改**：與 #5 同一個 tooltip 元件處理，多顯示「成交量」欄
- **影響**：[candlestick-chart.tsx](web/src/components/dashboard/candlestick-chart.tsx)（同 #5）

### 7. 全部繁體中文

- **使用者決定**：**本輪跳過**，目前 grep 沒看到簡體，暫不需處理

### 8. 日 K 下方留白過大

- **現況**：chart 容器 `h-[560px]`，主圖 + volume 沒填滿整個容器，下方有空白塊
- **修改**：高度從 560 → **440 ~ 480px**（實測決定），volume 區比例維持 16% 不變
- **影響**：[candlestick-chart.tsx](web/src/components/dashboard/candlestick-chart.tsx)

### 9 + 10. 底部三 panel 佈局重排

- **現況**：底部一列三等分（K線型態 ｜ 多週期 ｜ AI 劇本）
- **修改**（**使用者確認 = (A)**）：底部變雙欄
  - **左 70%**：K線型態 與 多週期·量價 **並排**（占滿主圖下方寬度）
  - **右 30%**：AI 劇本 panel（與上方右欄垂直對齊）
- **影響**：[dashboard-page-client.tsx](web/src/components/dashboard/dashboard-page-client.tsx) 底部 grid 改 `grid-cols-[minmax(0,1fr)_360px]`，左格內再分 2 col

### 11. K 線型態欄位

- **現況**：表格欄寬自動，「狀態」欄太窄導致 `成立 / 未成` badge 文字被擠成兩行
- **修改**：
  - 「狀態」欄固定寬度 ~64~80px，badge 加 `whitespace-nowrap`
  - 「說明」欄改用舊 Streamlit 版的完整描述文字（**使用者確認來源 = [src/ui/pages/dashboard.py](src/ui/pages/dashboard.py)**），單格放不下時允許多行
- **影響**：[dashboard-page-client.tsx](web/src/components/dashboard/dashboard-page-client.tsx) 的 `PatternsPanel` + 需要參照 [src/ui/pages/dashboard.py](src/ui/pages/dashboard.py) 把描述文字搬過來

### 12. (?) tooltip 點擊/hover 都沒反應

- **根因**：[help-tooltip.tsx:11](web/src/components/dashboard/help-tooltip.tsx) 用 HTML `title` 屬性 —— 只有 hover 1+ 秒才會出現原生 tooltip，視覺醜且點擊無效
- **修改**：改用 **Radix UI Tooltip**（已透過 `@radix-ui/react-*` 系列載入，但目前未安裝 `@radix-ui/react-tooltip`，需 `pnpm add` 補裝）
  - 觸發（**使用者確認**）：**hover 為主**，點擊也應該觸發（行動裝置 / 無障礙）
  - 樣式：與其他 Radix dialog 一致
- **影響**：
  - [help-tooltip.tsx](web/src/components/dashboard/help-tooltip.tsx)（重寫）
  - 需要在 root layout 加 `<Tooltip.Provider>` 包覆，或讓元件自己包

### 13. 壓縮到 1.2 ~ 1.5 個視窗高度

- **目標**（**使用者確認**）：1.2 ~ 1.5 視窗，**不需強塞到 0 捲動**
- **手段**：#1（側欄縮）+ #2（右欄上移、緊湊化）+ #3（MA 細化釋出 chart 空間）+ #8（chart 高度縮）+ #9-10（底部重排）綜合作用
- **參考解析度**：使用者 monitor 2560×1217；最低應支援 1440×900 PC

### 風險 / 注意事項

| # | 風險 | 緩解 |
|---|---|---|
| 5/6 | crosshair tooltip 在 SSR 環境可能 hydration mismatch | tooltip DOM 用 `useEffect` 內動態 `appendChild`，避免 SSR 渲染 |
| 12 | Radix Tooltip 需要 Provider 包覆 | 在 `web/src/app/layout.tsx` 的 `ThemeProvider` 內加一層 `Tooltip.Provider delayDuration={150}` |
| 1 | 側欄縮窄後現有 `text-slate-200` 等樣式可能跑版 | 改完手動驗 + 可能加 vitest snapshot |
| 4 | `MarketSwitcher` 改動可能影響 [tests/components/market-switcher.test.tsx](web/src/tests/components/market-switcher.test.tsx) | 預期斷言會壞，需同步更新測試（行為斷言為主、class 斷言放寬） |
| 全 | dashboard-page-client 大幅重排可能造成 `tsc / vitest / build` 連鎖反應 | 每改完一塊跑一次三件驗證；分小 commit 落地 |

### 驗證計畫

依序執行直到綠燈：

1. `web/` `npx tsc --noEmit` — 0 errors
2. `web/` `npm test`（vitest）— 全綠（可能需同步更新 market-switcher 測試）
3. `web/` `pnpm build` — 通過（< 1 min）
4. 手動：`run_quanttraderV2.bat` 開瀏覽器，逐項目視比對
5. backend pytest：理論上不會被影響，但跑一次確認：`./.venv/Scripts/python.exe -m pytest tests/test_api/` — 25 passed
